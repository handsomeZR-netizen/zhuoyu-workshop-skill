# 提示词模式

这些提示词可以直接复制到 Codex 或其他 AI 工具中。使用前把方括号内容替换成真实信息。

## 项目诊断

```text
使用琢玉工坊帮我诊断一个学生项目。

项目名称：[项目名称]
项目类型：[大创 / 竞赛 / 课程设计 / 毕设 / 科研小项目]
技术栈：[技术栈]
已有材料：[代码、README、报告、截图、日志、数据]
目标用途：[答辩 / 结题 / GitHub 展示 / 申报]
限制条件：[不能虚构的数据、不能修改的内容]

请输出：
1. 项目一句话定位；
2. 真实成果和薄弱环节；
3. 可包装的亮点；
4. 还需要补齐的证据；
5. 下一步交付物清单。
```

## 答辩 PPT

```text
使用琢玉工坊为我的项目生成答辩 PPT 方案。

项目名称：[项目名称]
答辩时长：[例如 5 分钟]
目标受众：[评委老师 / 任课老师 / 竞赛评委]
已有材料：[报告、截图、系统功能、演示视频]
希望页数：[例如 8 页]
限制条件：[不能虚构上线、获奖、论文、用户量]

请输出：
1. 每页标题；
2. 每页核心结论；
3. 推荐图示或截图；
4. 讲稿要点；
5. 演示路线；
6. 评委可能提问和回答。
```

## README 优化

```text
使用琢玉工坊优化我的 GitHub README。

项目名称：[项目名称]
项目用途：[一句话说明]
技术栈：[技术栈]
当前仓库情况：[已有 README / 没有 README / 安装步骤不完整]
希望风格：[工程简洁 / 学生项目展示 / 竞赛作品展示]
限制条件：[不能写不存在的在线 Demo、测试覆盖率、用户量]

请输出一个完整 README 草稿，包括：
项目简介、功能特性、技术栈、架构说明、快速开始、截图位置、项目结构、已知限制和路线图。
```

## 报错诊断

```text
使用琢玉工坊按 Codex 安全工作流诊断这个报错。

项目技术栈：[技术栈]
报错信息：[完整错误日志]
复现步骤：[如何触发]
相关文件：[文件路径]
限制条件：[不能修改哪些文件 / 不能重构哪些模块]

请先分析原因，不要直接改代码。输出：
1. 最可能原因；
2. 需要检查的文件；
3. 最小修改方案；
4. 验证命令；
5. 回滚方式。
```

## 结题报告

```text
使用琢玉工坊帮我整理结题报告。

项目名称：[项目名称]
项目来源：[大创 / 课程 / 竞赛 / 毕设]
已有成果：[系统功能、实验结果、代码、截图、报告草稿]
缺失材料：[没有的材料也要写清楚]
学校要求：[字数、章节、格式、评分点]
限制条件：[不能虚构成果、不能夸大应用效果]

请输出：
1. 报告目录；
2. 每章写作要点；
3. 成果矩阵；
4. 证据补齐清单；
5. 风险表述和不足展望。
```
