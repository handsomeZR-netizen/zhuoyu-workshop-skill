# 示例：GitHub README 优化

## 用户输入

```text
使用琢玉工坊优化我的 README。

项目名称：LabReserve
用途：实验室预约和设备管理系统
技术栈：Next.js、Supabase、Tailwind CSS
当前问题：README 只有启动命令，没有项目介绍和截图位置
限制条件：没有在线 Demo，不要写部署地址。
```

## README 应包含

- 项目简介：解决什么预约管理问题。
- 功能特性：预约、审批、设备管理、通知、权限。
- 技术栈：前端、后端、数据库、认证。
- 快速开始：环境变量、安装、启动。
- 截图位置：用占位说明提醒用户补图。
- 项目结构：只列关键目录。
- 已知限制：本地演示版本，通知和审计能力待完善。
